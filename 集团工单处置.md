# 集团工单

## 一、云 SOC 工单处置要求

进入【**全流量**】 → 【**高危 IP 工单**】，选择近 7 日进行查询，核查相关 IP 是否存在安全告警/攻击记录。

1. 对所有待核查 IP **逐一查询近 7 日记录**。

2. 对查询到有攻击/告警记录的 IP：
   - ✅ 按下方模板填写处置信息
   - ✅ **执行封禁操作**
   - ✅ 写入表格【**威胁情报**】

3. 对查询无记录的 IP：
   - ❌ 无需封禁
   - ❌ 无需写入【**威胁情报**】

4. **所有 IP/域名均需统一导入情报库。**
   > 情报库导入流程：【**知识库**】 → 【**情报库**】 → 【**情报管理**】 → 【**情报导入**】
   > 情报源选择【**IP及域名黑名单情报**】，类型按实际情况选择。

5. 全部处置完成后，在表格【**工单处置记录**】中补充本次处置记录。

---

## 二、高危 IP 预警工单处置要求

**集团综合调度接单后，按照工单要求的日期完成处置并提交结果。**

1. 下载工单附件，**筛选出带颜色标注的 IP**。
2. 进入【**全流量**】 → 【**威胁中央企业数据上报-勿删**】。
3. 按工单要求日期设置并保存筛选条件，查询相关 IP 的攻击/告警记录，便于后续复核和追溯。
4. **剩余处置流程同上方云 SOC 工单第 2 步至第 5 步执行。**

---

## 三、工单处置记录填写示例

**有攻击记录时：**
> 匹配全流量近7日内相关安全告警涉及攻击IP 3个（176.65.148.93、64.89.162.139、45.142.154.10），已经进行封禁处置，所有IP已经加入到全流量情报库中进行监测。

**无攻击记录时：**
> 匹配全流量近7日内相关安全告警涉及攻击IP 0个，所有IP已经加入到全流量情报库中进行监测。
